# SGU-SAIC-SGMW-BYD-Linux-radar_to_csv_gui_2.2 说明  

## 1. 概述
将AUTOSAR版SGU/IMU目标数据，.bag数据转CSV（***存入`GTs/[输出文件夹]/`***,或者***存入`IMUs/[输出文件夹]/`***），筛选动态目标（可以选择全部目标，详细见bash start）（``dynFlg≠0且objID>0``），支持SAIC/SGMW/BYD等车，基于ROS1+Qt5的图形化工具体系。  


## 2. 环境与启动  
- 环境：Ubuntu 18.04+（推荐18.04/20.04），依赖ROS Noetic、Qt5≥5.5、arbe_msgs包；  
需Bag目录读权限、输出目录写权限。  

- 启动：  
  ```bash
    bash start  # 自动启roscore（未运行则启），加载环境后显界面
  ```  


## 3. 操作  
- **参数配置**： 
  - Bag文件夹：含.bag的目录，Bag需含`arbe_msgs/wfSguRawData`消息；  
  - 车辆编号：字母数字（2-20字符，用于CSV命名）；  
  - 后缀：默认GT；  
  - 输出文件夹：默认2025_10_05。(***改为Bag实际录制时间即可,存入`GTs/[输出文件夹]/`***)



## 4. 数据与规则  
- **雷达话题**：4个WF角雷达话题（如`/wf/corner_radar/sgu_data_1`对应左前WFRAFL）。  
- **筛选**：仅保留动态目标（dynFlg≠0且objID>0）。  
- **命名**：  
  - Bag：`[前缀]_YYYY-MM-DD-HH-MM-SS_[后缀].bag`；  
  - CSV：`[车辆编号]_[雷达位置]_[日期]_[时间]_[后缀].csv`  


## 5. CSV_CUT.py - CSV分割工具

### 概述
`CSV_CUT.py` 是一个附属Python脚本，用于对主程序生成的CSV文件进行智能分割，主要解决以下问题：
1. **满足Bosch激光雷达KPI要求**：将长时数据分割为符合测试标准的时间段
2. **处理大文件不便查看**：分割过大的CSV文件，便于数据分析和查看


### 使用方法
```bash
python3 CSV_CUT.py
```

### 操作流程
1. **选择源CSV文件夹**：选择需要分割的CSV文件夹
2. **配置分割参数**：
   - 分割模式：固定时长分割/自定义时间段
   - 分割时长（如：300秒）
   - 输出目录
3. **执行分割**：
   - 脚本读取CSV文件
   - 根据时间戳进行分割
   - 生成多个子CSV文件

### 输出文件命名
分割后的文件按以下格式命名：
```
[车辆编号]_[雷达标号]_[数据段起始日期]_[数据段起始时间]_[后缀].csv
```

## 6. IMU_radar_to_csv
针对IMU数据，将IMU数据与雷达数据进行匹配，生成CSV文件，并保存到指定目录。
